# StoryLoom 项目方案 (核心功能版)

## 1. 核心理念

构建一个专注于**高质量视频生成**的核心工具。用户无需登录，即可直接访问网站，通过文字或图片快速创作 4K 视频。重点是提供流畅的创作体验和惊艳的视觉效果。

## 2. 核心功能模块 (简化后)

1.  **统一创作界面:**
    *   一个简洁的中心区域，用户可以在此直接输入文字 Prompt 或上传图片。
    *   提供清晰的模式切换按钮（“文生视频” / “图生视频”）。

2.  **Prompt 工作台:**
    *   **Prompt 优化器:** 用户输入初步想法后，可以点击“优化”按钮。后台将调用强大的语言模型（如 GPT-4, 智谱 GLM, Claude 等）对 Prompt 进行分析、扩写和润色，提供更具创意和细节的版本供用户选择。
    *   **Prompt 历史:** 使用浏览器本地存储 (Local Storage) 记录用户最近使用过的 Prompt，方便查找和复用。

3.  **高级参数面板:**
    *   用户可以轻松调整关键参数，例如：
        *   **视频风格:** 电影、动漫、写实、水彩、科幻等。
        *   **画面比例:** 16:9 (横屏), 9:16 (竖屏), 1:1 (方形)。
        *   **运动强度:** 控制镜头移动或画面变化的幅度。

4.  **任务状态追踪器:**
    *   用户提交生成请求后，页面上会有一个实时的任务列表。
    *   清晰地显示每个视频任务的状态：`排队中` -> `生成中` -> `已完成` / `失败`。
    *   这个列表的状态也保存在浏览器会话中。

5.  **作品展示与下载:**
    *   生成完成的视频会直接出现在页面的结果区域。
    *   提供在线预览播放功能。
    *   提供不同分辨率的下载选项，特别是 **1080p (标准)** 和 **4K (超清)**。

## 3. AI 模型选型 (已更新)

1.  **文生视频 / 图生视频 (直接生成 4K):**
    *   **智谱 AI (CogVideo):** 能够直接输出 4K 视频，极大地简化了工作流程，是此项目的首选。
    *   **商业 API (备选/未来扩展):** Runway Gen-2, Pika Labs。
    *   **开源模型 (备选/未来扩展):** Stable Video Diffusion。
    *   **未来关注:** OpenAI Sora, Kling (快手)。

2.  **Prompt 优化 (关键功能):**
    *   **智谱 AI (GLM 系列):** 作为国内顶尖的语言模型，对于中文 Prompt 的理解、优化和扩写能力非常出色，是此功能的绝佳选择。
    *   **GPT-4o / GPT-4 Turbo (OpenAI):** 强大的通用语言理解和生成能力。
    *   **Claude 3 Opus (Anthropic):** 尤其擅长创意性写作和细节描述。
    *   **Google Gemini:** 优秀的多模态理解能力。

3.  **视频超分辨率 (如果主模型不支持 4K 或需要更高质量):**
    *   **开源模型:** Real-ESRGAN, SwinIR。
    *   **商业方案:** Topaz Video AI。
    *   *注：如果智谱 CogVideo 能直接输出高质量 4K，则此步骤可省略。但如果需要提供 1080p 下载选项，可能需要使用 FFmpeg 进行降采样。*

## 4. 直接生成 4K 视频的工作流程 (基于智谱 CogVideo)

这个方案的核心是，我们将所有的复杂性都交给了智谱 AI 的模型，我们的后端只负责调用和接收结果。

#### 后端工作流程 (大幅简化)

1.  **接收请求**:
    *   前端将用户的 Prompt、选择的风格、画面比例等参数发送到后端 API。

2.  **创建并分派异步任务**:
    *   后端 API 接收到请求后，立即创建一个任务，并将其放入 Celery 任务队列。
    *   任务的初始状态为 `排队中`。
    *   API 立刻返回一个任务 ID 给前端，以便前端可以轮询任务状态。

3.  **执行视频生成 (Celery Worker)**:
    *   后台的 Celery Worker 从队列中获取任务。
    *   更新任务状态为 `生成中`。
    *   **核心步骤**: Worker 直接调用智谱 AI 的 CogVideo API/SDK。
    *   在调用时，会传入一个关键参数，明确要求输出 **4K 分辨率** (例如，可能会有类似 `resolution: "4k"` 或 `size: "3840x2160"` 的参数)。
    *   这个 API 调用会是一个**长时间等待**的过程。Worker 会一直等待，直到 CogVideo 模型完成处理并返回结果。

4.  **处理结果**:
    *   CogVideo API 返回处理结果，这通常是一个包含 4K 视频文件的 URL。
    *   Worker 接收到这个 URL，并将其保存到我们的数据库（或一个简单的文件/缓存中），与任务 ID 关联起来。
    *   更新任务状态为 `已完成`。
    *   *如果需要提供 1080p 下载选项，可以在此步骤使用 FFmpeg 对 4K 视频进行快速降采样，生成一个 1080p 版本并存储。*

5.  **完成与清理**:
    *   任务完成。由于没有中间文件（如帧图片），所以也**不需要任何清理工作**。

## 5. 需要考虑的问题

1.  **API 成本**: 直接生成 4K 视频的 API 调用费用，几乎肯定会比生成标清/高清视频**昂贵得多**。这是个人项目需要重点衡量的。
2.  **生成时长**: 尽管流程简化，但单次任务的等待时间可能会非常长。需要在前端做好用户预期管理，例如提供一个更精确的进度条（如果 API 支持的话），或者允许用户关闭页面，后续再回来查看结果。
3.  **提供多种分辨率下载**: 如果模型只输出一个 4K 视频，但我们想为用户提供更小的 1080p 下载选项，我们需要在视频生成后，用 FFmpeg **降采样 (Downscaling)** 创建一个 1080p 版本。这个操作非常快，而且消耗资源很少。
